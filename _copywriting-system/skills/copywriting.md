# copywriting

## Когда использовать

Когда нужно написать, переписать или улучшить коммерческий текст: hero, оффер, преимущества, CTA, FAQ, блоки доверия, тексты страниц, лид-магниты, объявления и сценарии заявки.

## Входные документы

Сначала проверить, есть ли:

- `POSITIONING_FINAL.md`
- `TARGET_AUDIENCE.md`
- `SITE_ARCHITECTURE.md`
- нужный `PAGE_*.md`
- `DESIGN_SYSTEM.md`, если текст влияет на визуальную структуру

Если документов нет, не выдавать финальную уверенную подачу. Писать как гипотезу и помечать пробелы.

## Алгоритм

1. Определи аудиторию, задачу страницы и главный следующий шаг.
2. Сформулируй одну центральную мысль: почему клиент должен оставить заявку именно здесь.
3. Переведи свойства услуги в последствия для клиента: деньги, время, риск, контроль, скорость, спокойствие.
4. Используй язык клиента, а не язык компании.
5. Каждый блок должен двигать аргумент вниз по странице: проблема -> решение -> доказательство -> действие.
6. CTA писать как действие с понятным результатом, не как пустую кнопку.

## Правила текста

- Ясность важнее креатива.
- Конкретика важнее красивых прилагательных.
- Один блок - одна мысль.
- Заголовок должен быть понятен без подзаголовка.
- Подзаголовок уточняет, а не повторяет H1.
- Не обещать результат, если нет доказательств.
- Не использовать фразы "индивидуальный подход", "команда профессионалов", "широкий спектр услуг" без конкретизации.

## Выходной формат

Для страницы:

```text
H1:
Подзаголовок:
Primary CTA:
Secondary CTA:
Блок 1:
Блок 2:
FAQ:
Финальный CTA:
Что требует проверки:
```

Для редакторской правки:

```text
Было:
Стало:
Почему лучше:
Риск/допущение:
```
