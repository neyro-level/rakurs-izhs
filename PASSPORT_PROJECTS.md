# PASSPORT_PROJECTS — Сайт Ракурс ИЖС

## 1. Что это

Рабочий Astro-сайт проекта Ракурс ИЖС.

Проект собирается по методологии Чиркова: исследование, архитектура, дизайн-система и техническая сборка на Astro.

## 2. Тип проекта

- Клиентский Astro-проект АМС.
- Рабочая папка: `C:\Users\User\Desktop\Сборка АМС\Ракурс ИЖС\Сайт Ракурс ИЖС`.
- Режим работы: Фаза 4, сборка и доработка сайта по утверждённым документам.

## 3. Стек

- Astro.
- Tailwind CSS v4.
- TypeScript strict.
- React islands для интерактива.
- pnpm.
- GitHub → AMS-сервер.

Команды:

```bash
pnpm install
pnpm dev
pnpm build
```

## 4. Что читать первым

1. `PASSPORT_PROJECTS.md`.
2. `WORKLOG.md`.
3. `PROJECT_JOURNAL.md`.
4. `PAGE_HOME.md` — **единственный источник правды по структуре и текстам главной страницы** (v2.0, 10 блоков).
5. `DESIGN_SYSTEM.md`.
6. Глобальные Astro/Codex-правила из настроек Codex и AI-SYSTEM.
7. Актуальный файл блока/страницы, который меняется.

Локальные копии копирайтинговых скиллов удалены. Активные скиллы запускаются через глобальный `C:\Users\User\.codex\skills`.

## 5. Источники правды

- `PAGE_HOME.md` v2.0 — **единственный источник правды по структуре, текстам, CTA, FAQ и SEO главной страницы**. Аннулирует все предыдущие версии и `SITE_ARCHITECTURE.md` (удалён).
- `PROJECT_JOURNAL.md` — журнал утверждённых решений проекта.
- `DESIGN_SYSTEM.md` — дизайн-система.
- `MARKETING_RESEARCH.md`, `PAINS.md` — исследовательская база (evidence, не структура).
- `SEO_WORDSTAT_PERM_2026-06-11.md` — семантика Wordstat по Перми.
- Глобальные Astro/Codex-правила — единственный источник методики технической сборки.

## 6. Локальные правила

- Не начинать сборку без актуального брифа страницы.
- Если код и бриф расходятся после прямой правки пользователя, сначала обновить бриф/журнал как `as-built`.
- Один блок = один `.astro` файл.
- Route-файл держать тонким.
- Цвета и типографику брать из `DESIGN_SYSTEM.md` и `src/styles/global.css`.
- Перед финалом запускать `pnpm build` и проверять страницу в браузере.

## 7. Журналирование

После содержательной правки обновлять:

1. `PROJECT_JOURNAL.md`;
2. `WORKLOG.md`, если он создан;
3. соответствующий `PAGE_*.md` / стратегический документ.

Локальный `AGENTS.md` в проекте не создавать. Единственный `AGENTS.md` находится в глобальных настройках Codex: `C:\Users\User\.codex\AGENTS.md`.
