# PASSPORT_PROJECTS — Сайт Ракурс ИЖС

## 1. Что это

Рабочий production-контур Astro-сайта проекта Ракурс ИЖС.

Документационный контур проекта очищен: актуальные правила и решения держим только в живых проектных документах, без локальных методических дублей.

## 2. Тип проекта

- Клиентский Astro-проект АМС.
- Рабочая папка: `C:\Users\User\Desktop\Сборка АМС\Ракурс ИЖС\Сайт Ракурс ИЖС`.
- Режим работы: Фаза 4 — production support, дизайн-шлифовка и подготовка роста через SEO-страницы.

## 3. Стек

- Astro.
- Tailwind CSS v4.
- TypeScript strict.
- React islands для интерактива.
- pnpm.
- GitHub → AMS-сервер.

Команды:

```bash
pnpm install
pnpm dev
pnpm build
```

## 4. Что читать первым

1. `PASSPORT_PROJECTS.md`.
2. `WORKLOG.md`.

Локальные копии копирайтинговых скиллов удалены. Активные скиллы запускаются через глобальный `C:\Users\User\.codex\skills`.

## 5. Источники правды

- `PAGE_HOME.md` v2.0 — **единственный источник правды по структуре, текстам, CTA, FAQ и SEO главной страницы**.
- `PROJECT_JOURNAL.md` — единый журнал утверждённых решений проекта.
- `SITE_DEVELOPMENT_STRATEGY.md` — активный стратегический документ по архитектуре роста сайта. Не дублирует `PAGE_HOME.md`, а дополняет его на уровне SEO-страниц и roadmap развития.
- `DESIGN_SYSTEM.md` — дизайн-система.
- `MARKETING_RESEARCH.md`, `PAINS.md` — исследовательская база (evidence, не структура).
- `SEO_WORDSTAT_PERM_2026-06-11.md` — семантика Wordstat по Перми.
- `src/pages/index.astro` и активные файлы `src/pages/_home/01-10` — текущая as-built реализация, которая должна соответствовать `PAGE_HOME.md`.
- Глобальные Astro/Codex-правила — единственный источник методики технической сборки.

## 6. Локальные правила

- Не начинать сборку без актуального брифа страницы.
- Если код и бриф расходятся после прямой правки пользователя, сначала обновить бриф/журнал как `as-built`.
- Один блок = один `.astro` файл.
- Route-файл держать тонким.
- Цвета и типографику брать из `DESIGN_SYSTEM.md` и `src/styles/global.css`.
- Не создавать локальные `README.md`, дублирующие стратегические документы и методические папки вроде `00_CORE/`; если решение уже зафиксировано, обновлять существующий source-of-truth.
- Перед финалом запускать `pnpm build` и проверять страницу в браузере.

Локальный `AGENTS.md` в проекте не создавать. Единственный `AGENTS.md` находится в глобальных настройках Codex: `C:\Users\User\.codex\AGENTS.md`.
