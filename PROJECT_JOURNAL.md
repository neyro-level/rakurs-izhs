### 2026-06-14 — Analytics-модуль переносим из «Союз Ростов» выборочно, а не целиком

**Статус:** выполнено и зафиксировано как рабочий production-подход.

**Решение:**

- Из проекта `Союз Ростов (AMS)` для `Ракурс ИЖС` перенесён только аналитический контур, который действительно уместен для текущего сайта:
  - `@astrojs/partytown` как transport-слой;
  - consent-aware bootstrap Яндекс.Метрики;
  - единый `src/lib/analytics.ts` для event tracking.
- Перенос сделан не как буквальная копия проекта `Союз Ростов`, а как адаптация под канон `13_METRICS_ANALYTICS_MODULE.md`.
- Метрика не запускается до consent.
- Event tracking централизован и не размазан хаотичными `ym(...)` по компонентам.

**Что именно отслеживается:**

- клики по CTA, открывающим модалку;
- открытие и закрытие модалки;
- попытка отправки формы;
- успешная отправка и ошибки формы;
- клики по телефону;
- клики по мессенджерам;
- открытие FAQ;
- глубина скролла.

**Что сознательно НЕ переносим сейчас из `Союз Ростов`:**

- `ClientRouter` и client-side transitions;
- `astro-loading-indicator`;
- `@playform/compress`;
- `astro-icon`;
- другие infra-улучшения, не дающие прямой пользы текущему one-page сайту на этом этапе.

**Почему:**

- для `Ракурс ИЖС` сейчас важнее измеримость конверсии и аккуратный consent-layer, чем расширение навигационного или build-слоя;
- лишние инфраструктурные переносы увеличивают риск regression без явной коммерческой пользы.

**Источник правды:**

- `astro.config.mjs`
- `src/lib/analytics.ts`
- `src/layouts/BaseLayout.astro`
- `src/components/shared/CookieBanner.tsx`
- `src/components/shared/ModalShell.tsx`
- `src/components/shared/RequestModal.tsx`

**Важно:**

- Analytics should never break UX: если аналитика не готова или пользователь не дал consent, сайт работает без ошибок и без падения интерфейса.
- Это решение не отменяет будущий перенос других infra-модулей из `Союз Ростов`, но только по отдельной задаче и после оценки пользы.
