## 1. Статус данных

### Верифицировано

- RACURS не строит сам, а помогает клиенту собрать безопасный маршрут к дому: участок, ипотека, подрядчик, смета, реальные расходы.
- Главный оффер: дом по ипотеке 6% в Перми без ошибки в участке, подрядчике и смете.
- Главный продукт лендинга: **разбор сценария дома**.
- Для клиента сопровождение заявлено без отдельной оплаты.
- Модель оплаты должна объясняться заранее: вознаграждение RACURS выплачивает подрядчик после сделки.
- Главный риск коммуникации: клиент может решить, что RACURS просто ведет его к подрядчику, который платит комиссию.
- ЦА: семьи и покупатели из Перми, которые хотят дом по семейной ипотеке 6% или уже сравнивают подрядчиков и сметы.
- Архитектура лендинга зафиксирована в `site_architecture_racurs_landing.md`.
- Фирменный синий RACURS на корпоративном сайте: `#0874C4` / `#0474C4`.

### Гипотеза

- Темно-синий первый экран повысит ощущение статуса, серьезности и контроля.
- Контрастная визуальная система будет сильнее полностью светлого лендинга, потому что услуга сложная и связана с дорогим решением.
- Сочетание темного Hero и светлых доказательных блоков снизит риск монотонности без потери доверия.
- Manrope даст продукту более современную и собранную подачу, чем прямое копирование Rubik с корпоративного сайта.

### Требует проверки

- Есть ли отдельный официальный брендбук RACURS с точными HEX, правилами логотипа и типографикой.
- Есть ли реальные фото эксперта, объектов, участков, документов и смет, которые можно использовать на сайте.
- Нужен ли отдельный `PAGE_HOME.md` / `PAGE_LANDING.md`; сейчас источником правды по структуре является `site_architecture_racurs_landing.md`.

# § 1. Концепция системы

## Название системы

**RACURS Navy Control**

## Формула направления

**Контрастная экспертная система для безопасного маршрута к дому.**

Это не сайт строительной компании, не каталог домов и не ипотечный лендинг. Визуальный язык должен показывать: **RACURS помогает клиенту принять дорогое решение без хаоса, слепых зон и давления подрядчика**.

## Главная визуальная идея

**Темно-синий экспертный старт + светлая доказательная структура + контрастные секции доверия.**

Первый экран должен выглядеть сильнее и современнее корпоративного сайта RACURS, но оставаться узнаваемым по синему цвету, аккуратности и недвижимости-контексту.

## Эталоны

| Эталон | Что берем |
|---|---|
| **Т-Банк** | уверенность, контраст, понятные финансовые сценарии |
| **Яндекс Недвижимость / Домклик** | привычность для рынка недвижимости, карточки, сценарность |
| **Vercel** | чистая типографика, темные секции, технологичная аккуратность |
| **Stripe** | финансовая серьезность, визуальная логика сложных продуктов |
| **Mercury** | деловая премиальность без перегруза |
| **RACURS corporate** | брендовый синий, недвижимость-контекст, локальная узнаваемость |

## Визуальный язык

Темно-синий контраст, светлая аналитическая структура, карточки проверки, таблицы сравнения, маршрутные схемы и рабочие документные блоки.

Лендинг должен выглядеть как **экспертная панель принятия решения**, а не как витрина коттеджей.

## Интонация

- Уверенная.
- Деловая.
- Спокойная, но не сонная.
- Контрастная, но не агрессивная.
- Объясняющая, но не перегруженная.
- Без инфобизнес-энергии.
- Без образа “мы сейчас построим вам дом мечты”.

## Для кого

Для семей и покупателей из Перми, которые хотят дом, но не готовы самостоятельно разбираться в участке, ипотеке, подрядчике, смете и реальных расходах.

Эта аудитория не ищет просто красивую картинку. Ей нужно почувствовать: **здесь есть порядок, проверка, критерии и понятный следующий шаг**.

# § 3. Типографика

## 3.1. Основной шрифт

**Manrope**

Причина выбора: хорошо работает в русскоязычных интерфейсах, выглядит современно, мягче Inter, но сохраняет деловую собранность. Для недвижимости и семейной аудитории Manrope лучше, чем слишком техническая типографика.

## 3.2. Брендовый fallback

**Rubik**

Rubik используется на корпоративном сайте RACURS. Его можно оставить fallback-шрифтом или использовать как альтернативу, если владелец бренда захочет сохранить более прямую связь с текущим сайтом.

Рекомендуемый CSS:

```css
font-family: "Manrope", "Rubik", system-ui, -apple-system, BlinkMacSystemFont, "Segoe UI", sans-serif;
```

## 3.3. Веса

- 400 Regular: основной текст.
- 500 Medium: кнопки, бейджи, навигация.
- 600 Semibold: H3, карточки, важные фразы.
- 700 Bold: H2, крупные смысловые заголовки.

## 3.7. Правила применения

- H1 на Hero может быть белым на темно-синем фоне.
- В Hero допустимо выделять одну смысловую фразу голубым, например “без ошибки”.
- H2 по левому краю. Центрирование только в финальном CTA.
- Абзацы не шире 58-65ch.
- Lead не шире 720px.
- В блоках “смета”, “риски”, “модель оплаты” текст должен быть коротким и структурным.
- Не использовать декоративные шрифты.
- Не использовать serif.
- Не делать весь текст жирным.
- Не использовать капс для длинных фраз.
- Не масштабировать шрифт через viewport width.

## 6.3. Принципы layout

### 1. Темный Hero как главный визуальный вход

Первый экран должен быть самым сильным визуальным блоком сайта.

Структура:

- слева: оффер, подзаголовок, CTA, trust-row;
- справа: panel “Разбор сценария дома”;
- фон: глубокий синий;
- поверх: тонкая сетка, радиальный свет, мягкие карточки.

### 2. Светлая доказательная середина

После Hero нельзя продолжать темный фон. Основная часть лендинга должна быть светлой, чтобы человек мог читать, сравнивать и возвращаться к аргументам.

### 3. Контрастные смысловые острова

Темный фон использовать в трех местах:

- Hero;
- блок “Кто платит RACURS”;
- финальный CTA.

### 4. Карточки вместо полотна текста

Ключевые смыслы оформлять как карточки:

- участок;
- ипотека;
- подрядчик;
- смета;
- реальные расходы;
- модель оплаты.

### 5. Табличность там, где клиент сравнивает

Для смет, критериев подрядчика и границ ответственности использовать таблицы или псевдотаблицы. Это усиливает ощущение контроля.

# § 7. Базовые принципы системы

## Принцип 1. Contrast with purpose

Контраст используется не для красоты, а для смысловой иерархии.

Темный фон:

- первый экран;
- модель доверия;
- финальный CTA.

Светлый фон:

- объяснение;
- проверка;
- сравнение;
- FAQ.

## Принцип 2. Route over catalogue

Визуально показываем не каталог домов, а маршрут:

**ипотека 6% -> участок -> подрядчик -> смета -> договор.**

Не делать главным блок “проекты домов”. Если проекты появятся, они вторичны.

## Принцип 3. Evidence first

Каждая сильная визуальная карточка должна содержать конкретику:

- что проверяем;
- зачем это важно;
- где риск;
- что получает клиент.

Без пустых преимуществ.

## Принцип 4. Calm control

Система должна снижать тревогу, но не быть блеклой. Тревогу снимают структура, контраст, таблицы, чек-листы, короткие формулировки.

## Принцип 5. No builder mimicry

RACURS не должен выглядеть как строительная компания.

Нельзя перегружать сайт:

- фасадами;
- проектами домов;
- рендерами;
- стройматериалами;
- касками;
- бригадами;
- обещаниями “построим”.

## Принцип 6. Transparent model

Блок “кто платит RACURS” должен быть визуально заметным, а не спрятанным в FAQ.

Это ключевой блок доверия.

## Принцип 7. One brand, separate product

Лендинг должен быть узнаваемо RACURS, но самостоятельнее и сильнее корпоративного сайта.

Сохраняем:

- синий;
- чистоту;
- недвижимость-контекст;
- аккуратность.

Усиливаем:

- контраст;
- структуру;
- экспертность;
- доказательность.
