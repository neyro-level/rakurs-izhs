# Методика Астро сайт

**Универсальная методика сборки B2B-сайтов на стеке Astro + Tailwind CSS v4 + TypeScript.**
Работает с любым AI-ассистентом: Claude Code, OpenAI Codex, Cursor, Kimi, Continue, Gemini.

**Версия:** 2.0
**Автор:** Андрей Чирков, АМС
**Дата:** 2 мая 2026

---

## Что это

Это **библиотека модулей**, не один статичный шаблон. На старте каждого нового клиентского проекта ты выбираешь нужные модули из библиотеки (с помощью AI-рекомендации), копируешь их в проект, заполняешь брифы, AI собирает финальный сайт.

Главное отличие от обычных стартеров: **ничего лишнего в каждом проекте**. Юр. сайт получает Editorial Trust пресет + блог + Authors. IT-агентство получает Tech Forward + анимации Premium. Стройка получает Chirkov B2B Premium + только marketing. У каждого свой stack под нишу.

## Как использовать (5 шагов)

### Шаг 1. Создай папку нового клиента
```bash
mkdir client-name && cd client-name
```

### Шаг 2. Скопируй ядро из библиотеки
Скопируй **всё содержимое `00_CORE/`** в корень нового проекта. Это базовые правила и конфиги, всегда одинаковые.

### Шаг 3. Заполни брифы
Скопируй шаблоны из `05_BRIEFS_TEMPLATES/` в `_briefs/` нового проекта. Заполни:
- `SITE_ARCHITECTURE.md` (обязательно)
- `PAGE_*.md` для каждой страницы (обязательно минимум один)
- `DESIGN_SYSTEM.md` если кастомный дизайн (опционально)

### Шаг 4. Запусти AI
Открой папку проекта в любом AI-ассистенте, отправь:
```
Прочитай AGENTS.md. В _briefs/ всё готово. Прорекомендуй stack и план файлов.
```

AI прочитает брифы, проанализирует нишу, **рекомендует подходящие модули** (дизайн-пресет, анимации, header, нужны ли блог/authors). Ты подтверждаешь или корректируешь.

### Шаг 5. AI собирает проект
После твоего «ок» AI копирует выбранные модули из библиотеки в проект и собирает Astro-сайт. Дальше push в GitHub → Vercel автоматически деплоит.

Подробнее в `00_CORE/METHOD.md`.

## Технологический стек

```
Astro 6.2+              — фреймворк (требует Node 22+)
Tailwind CSS v4         — стилизация (через @tailwindcss/vite)
TypeScript strict       — типизация
React 18                — только в островках
shadcn/ui               — UI-компоненты в островках
Lucide React            — иконки

Vercel                  — хостинг (фиксированный выбор)
GitHub                  — репозиторий каждого проекта
Sitepins                — CMS для клиентских правок (стандарт)
Яндекс.Метрика          — аналитика по умолчанию
```

| Программа | Минимальная версия | Где взять |

## Связь с основной АМС-методологией

Эта библиотека — реализация **Фазы 5** (сборка) АМС-методологии для Astro-стека.

| АМС-фаза | Где реализуется здесь |
|---|---|
| Фаза 1 (исследование, позиционирование) | Не входит — работа до этой методики |
| Фаза 2 (дизайн-система) | `01_DESIGN_PRESETS/` или своя через `05_BRIEFS_TEMPLATES/DESIGN_SYSTEM.template.md` |
| Фаза 3 (архитектура сайта) | `05_BRIEFS_TEMPLATES/SITE_ARCHITECTURE` + `PAGE` |
| Фаза 4 (Figma) | Опционально перед сборкой, не блокирует процесс |
| Фаза 5 (сборка) | **Вся эта библиотека** |
