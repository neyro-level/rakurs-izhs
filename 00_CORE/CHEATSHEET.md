# CHEATSHEET.md — Стартовые промпты

Готовые команды для копирования в любой AI-ассистент. Под каждый сценарий — отдельный промпт.

**Универсальная шапка любой сессии:**
```
Прочитай AGENTS.md в корне проекта. Дальше следуй его правилам без отклонений.
```

---

## Режим A — Greenfield (всё готово сразу)

**Когда использовать:** все source-документы готовы, нужен сайт за один проход.

```
Прочитай AGENTS.md в корне проекта. В _briefs/ заполнены все нужные документы.

Применяй Module Recommendation Logic из AGENTS.md раздел 4:
1. Извлеки параметры проекта из SITE_ARCHITECTURE.md
2. Подбери дизайн-пресет, функциональные модули, анимацию, header
3. Покажи рекомендацию с обоснованием
4. Дождись моего "ок"

После моего "ок":
- Перенеси документы из _briefs/ в финальные локации
- Скаффолди Astro-проект через npm create astro@latest
- Создай колокационную структуру pages/_pageName/ для каждой страницы
- Сгенерируй пустые блоки по числу секций в каждом PAGE_*.md
- Настрой path aliases в tsconfig.json
- Подключи Astro Compressor + Sitemap

Реальные тексты из брифов вставь сразу при сборке блоков.
```

---

## Режим B — Incremental: старт с одной страницей

**Когда использовать:** на руках только SITE_ARCHITECTURE + один PAGE-бриф.

```
Прочитай AGENTS.md. В _briefs/ лежат SITE_ARCHITECTURE.md и PAGE_HOME.md.

Применяй Module Recommendation Logic. Покажи рекомендацию stack.

После моего "ок":
- Скаффолди минимальный Astro-проект только со страницей home
- Используй колокационную модель (pages/_home/)
- Остальные страницы из SITE_ARCHITECTURE.md добавь как заглушки в навигацию Header (disabled или href="#")
- НЕ создавай роут-файлы для страниц без брифа
```

---

## Режим B — Incremental: добавление страницы

```
Прочитай AGENTS.md. Добавляю страницу [name]. Бриф в _briefs/PAGE_[NAME].md.

Применяй колокационную модель:
- pages/[name].astro как тонкий роут-файл
- pages/_[name]/ с PAGE-брифом и блоками

Активируй ссылку в навигации Header. Не трогай остальные страницы.
Перенеси бриф из _briefs/ в pages/_[name]/.
```

---

## Режим B — Incremental: добавление блог-модуля

```
Прочитай AGENTS.md. Подключаю блог к существующему проекту.

Скопируй из библиотеки 02_OPTIONAL_MODULES/blog/ нужные файлы.
Создай:
- src/content/config.ts со схемой blog collection
- src/pages/blog/index.astro (список статей)
- src/pages/blog/[slug].astro (страница статьи)
- src/pages/blog/category/[cat].astro
- src/pages/blog/tag/[tag].astro
- src/pages/rss.xml.ts

Добавь ссылку "Блог" в навигацию.
Не трогай существующие страницы.
```

## Универсальные промпты для проверок

### Проверка соответствия методике

```
Прочитай AGENTS.md. Пройди по чек-листу из раздела 16.
Покажи список нарушений с указанием файла и пункта чек-листа.
Не исправляй, только укажи.
```

### Проверка модульности блоков

```
Прочитай AGENTS.md (раздел 7 — Поблочная модульность).
Проверь все блоки в src/pages/_*/.
Покажи нарушения:
- cross-block импорты
- общие глобальные стили
- блоки, ссылающиеся на внешнее состояние
- неправильные имена файлов (не NN-PascalCase.astro)
```

### Аудит производительности

```
Прочитай AGENTS.md. Проведи аудит производительности:
- Размер JS-бандла
- Количество React-островков
- Наличие preload на Hero-изображении
- Lazy-loading на изображениях ниже fold
- Использование <Image /> вместо <img>

Покажи, что можно улучшить, без правок.
```

### Подготовка к сдаче клиенту

```
Прочитай AGENTS.md. Пройди полный чек-лист сдачи (раздел 16).
Проверь:
- Lighthouse будет 90+ Mobile / 95+ Desktop
- Все мета-теги на месте
- JSON-LD валиден
- Sitemap есть
- robots.txt корректен
- Mobile вёрстка ОК на 375px

Покажи отчёт по каждому пункту.
```
